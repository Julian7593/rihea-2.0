# 伴侣同步 MVP PRD

## 1. 文档信息

- 产品模块：伴侣家庭陪伴系统 V1
- 当前版本：V1.0
- 文档日期：2026-03-10
- 适用阶段：产品定义 / 交互设计 / 开发评审

## 2. 背景

妊安当前已经具备以下核心能力：

- 孕周与基础档案
- 每日情绪与睡眠打卡
- 胎动记录
- 风险评估
- 医疗支持与专家咨询
- `partnerSync` 开关占位

现阶段产品仍以孕妈单人使用为主，存在三个明显问题：

1. 孕妈需要解释自己的状态，沟通成本高。
2. 伴侣愿意参与，但不知道现在该做什么。
3. 当情绪风险升高时，系统只提醒孕妈本人，缺少外部支持闭环。

V1 不做泛家庭社区，而是先把产品升级为双人协作工具：让伴侣知情、能行动、不过度侵入隐私。

## 3. 产品目标

### 3.1 业务目标

- 提高伴侣绑定率
- 提高伴侣端任务完成率
- 提高孕妈对“被支持感”的主观评价
- 为后续家庭网络和 AI 陪伴能力建立账户关系与权限底座

### 3.2 用户目标

- 孕妈：少解释，多被理解，多被分担
- 伴侣：看得懂现在情况，知道下一步该做什么

### 3.3 成功标准

- 新开启伴侣同步的孕妈中，绑定成功率 >= 45%
- 已绑定用户中，伴侣 7 日活跃率 >= 35%
- 已绑定用户中，伴侣任务 7 日完成率 >= 40%
- 中高风险状态下，伴侣提醒触达率 >= 80%

## 4. 用户与场景

### 4.1 目标用户

- 主用户：孕期女性
- 协同用户：伴侣

### 4.2 核心场景

1. 孕妈今天状态差，但不想反复解释。
2. 伴侣想参与孕期照顾，但不知道如何帮忙。
3. 产检临近，需要有人陪同、提醒、准备问题单。
4. 情绪风险连续升高，需要伴侣介入支持。

## 5. 产品定位

一句话定位：

> 妊安 V1 是一个面向孕妈与伴侣的轻协作系统，把“记录”转化为“陪伴行动”。

核心原则：

- 共享摘要，不共享全部原始隐私
- 给伴侣动作，不给伴侣压力
- 风险提醒必须附带下一步建议
- 任何共享都以孕妈授权为前提

## 6. MVP 范围

### 6.1 必做

- 伴侣绑定
- 权限设置
- 伴侣首页
- 今日陪伴任务
- 共享产检与待办
- 风险提醒
- 沟通建议卡片

### 6.2 不做

- 多家庭成员协作
- 家庭动态流 / 相册 / 评论
- 实时聊天
- 家人查看详细症状、私密笔记、完整风险明细
- AI 自动生成长对话

## 7. 权限设计

V1 采用 3 档共享权限。

### 7.1 权限档位

1. 不共享
2. 仅共享状态摘要
3. 共享状态摘要 + 待办/产检

### 7.2 伴侣可见内容

在“仅共享状态摘要”下可见：

- 当前孕周
- 今日是否已打卡
- 今日状态标签
- 最近风险档位结果
- 今日系统建议动作

在“共享状态摘要 + 待办/产检”下额外可见：

- 下一次产检时间
- 双人待办列表
- 陪同准备清单

### 7.3 伴侣不可见内容

- 原始情绪分数
- 症状明细
- 私密备注
- 历史完整打卡记录
- 胎动逐条记录详情
- 专家咨询内容

## 8. 功能设计

### 8.1 功能一：伴侣绑定

目标：建立稳定的双人关系链路。

功能说明：

- 孕妈在“个人中心-伴侣同步”中开启功能
- 系统生成邀请码或分享链接
- 伴侣输入邀请码后完成加入
- 孕妈确认绑定
- 绑定成功后双方建立一对一关系

状态说明：

- 未开启
- 待绑定
- 已绑定
- 已解绑

### 8.2 功能二：权限设置

目标：降低隐私顾虑，提升开启率。

功能说明：

- 孕妈可随时切换 3 档共享权限
- 孕妈可关闭伴侣风险提醒
- 孕妈可设置“是否同步产检待办”

交互要求：

- 权限变更实时生效
- 每次变更给出可见范围提示

### 8.3 功能三：伴侣首页

目标：让伴侣一打开就知道“现在是什么情况，我该做什么”。

页面模块：

- 今日状态卡
- 今日陪伴任务
- 下次产检卡
- 沟通建议卡
- 最近进展卡

页面文案方向：

- 不展示“你对象今天 62 分”
- 展示“今天更需要安抚和陪伴，建议先做 1 件事”

### 8.4 功能四：今日陪伴任务

目标：把抽象支持转成具体行动。

任务来源：

- 今日打卡结果
- 最近 3 天趋势
- 风险评估结果
- 孕周阶段
- 是否临近产检

任务示例：

- 今晚陪散步 20 分钟
- 睡前一起做 3 分钟呼吸练习
- 帮她整理明天产检要问的问题
- 今天减少讲道理，先回应情绪
- 如果她愿意，陪她记录一次胎动

任务机制：

- 每日最多展示 3 条
- 至少 1 条可立即执行
- 伴侣可标记“已完成”
- 孕妈可看到“伴侣已完成”

### 8.5 功能五：共享产检与待办

目标：减少遗忘与推诿，提升协作感。

功能说明：

- 孕妈可以创建或确认下次产检
- 系统根据产检时间自动生成准备任务
- 双方可勾选完成状态

默认任务示例：

- 确认时间和地点
- 准备报告与证件
- 写下 3 个想问医生的问题
- 安排陪同与往返

### 8.6 功能六：风险提醒

目标：当系统识别中高风险时，把支持链路补上。

触发逻辑建议：

- 连续 3 天低情绪
- 睡眠连续偏低
- 风险评估达到中高风险
- 用户主动选择“我需要支持”

提醒对象：

- 孕妈本人
- 已绑定伴侣

伴侣收到的内容：

- 风险状态摘要
- 当前建议动作
- 是否建议陪同就医/咨询

禁止展示：

- 详细评分算法
- 过度惊吓文案

### 8.7 功能七：沟通建议卡

目标：降低伴侣“越帮越错”的概率。

形式：

- 一句话建议
- 不建议说的话
- 推荐的具体问句

示例：

- 建议说：“我先陪你把今天最难受的那件事理一下。”
- 不建议说：“你别想太多。”
- 推荐问句：“你现在更需要我陪着，还是帮你处理一件具体事情？”

## 10. 关键流程

### 10.1 首次开通流程

1. 孕妈进入“个人中心”
2. 点击“开启伴侣同步”
3. 查看共享说明并选择权限档位
4. 生成邀请码
5. 伴侣完成加入
6. 孕妈确认绑定
7. 进入伴侣首页

### 10.2 日常协作流程

1. 孕妈完成当天打卡
2. 系统生成状态摘要
3. 系统刷新伴侣任务
4. 伴侣查看任务并执行
5. 伴侣标记完成
6. 孕妈感知已被陪伴或支持

### 10.3 风险升级流程

1. 系统识别中高风险
2. 孕妈收到自我照护建议
3. 若已授权，同步推送给伴侣
4. 伴侣端展示“现在先做什么”
5. 必要时引导到专家咨询或医疗支持

## 11. 页面草图要求

### 11.1 孕妈端伴侣同步卡

核心元素：

- 当前状态：未开启 / 待绑定 / 已绑定
- 伴侣昵称
- 当前权限档位
- 邀请入口
- 权限修改入口
- 解绑入口

### 11.2 伴侣首页

核心模块顺序：

1. 今日状态
2. 今日先做这一件事
3. 其他可选任务
4. 下次产检
5. 沟通建议

### 11.3 风险提醒页

核心原则：

- 先写可做什么
- 再写为什么
- 最后给专业支持入口

## 12. 与现有能力的衔接

建议直接复用现有能力，不重新造轮子。

- 情绪与睡眠打卡：作为任务和状态摘要输入
- 风险评估：作为提醒触发器
- 胎动记录：只同步“是否建议陪伴记录”，不开放明细
- 医疗支持中心：作为风险升级后的承接页
- `partnerSync`：从设置开关升级为完整功能模块

## 14. 验收标准

### 14.1 产品验收

- 孕妈可独立完成开启、邀请、修改权限、解绑
- 伴侣可独立完成加入和查看首页
- 伴侣首页能稳定展示状态、任务、产检信息
- 风险提醒可按授权规则正确触发
- 不应泄露私密备注和原始明细

### 14.2 体验验收

- 首次开通 3 分钟内可完成
- 伴侣首页 5 秒内能理解“今天要做什么”
- 权限切换后用户能清晰知道哪些信息被共享

## 15. 风险与对策

### 15.1 隐私风险

问题：孕妈担心被过度看见。

对策：

- 默认只开启“状态摘要”
- 敏感数据默认不共享
- 强化权限说明

### 15.2 伴侣低活跃风险

问题：伴侣绑定后不打开。

对策：

- 首页极简
- 每天只给 1 到 3 条任务
- 任务必须短、具体、能完成

### 15.3 提醒过度风险

问题：系统把伴侣变成另一个压力源。

对策：

- 中高风险才触发升级提醒
- 文案避免指责和惊吓
- 允许孕妈关闭部分提醒

## 16. 后续版本预留

V2 预留方向：

- 家庭成员扩展
- 紧急联系人树
- 产检陪同排班

V3 预留方向：

- AI 伴侣教练
- 个性化沟通建议
- 风险场景自动陪伴 SOP

## 17. 一句话总结

V1 的本质不是“给伴侣看数据”，而是“把孕妈的状态转译成伴侣今天能完成的支持动作”。
