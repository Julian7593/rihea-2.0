# AI Agent 迭代规划

## 1. 当前系统定位

当前 AI 不是自由发挥型的全能 Agent，而是一个：

`规则优先 + 风险强拦截 + 检索增强 + LLM 受控生成`

的孕期/情绪健康场景内置助手。

当前已具备的核心能力：
- 主路由编排：统一入口、统一审计、统一响应结构
- 安全机制：输入预检 + 输出后检 + 高风险优先升级
- 子 Agent：以 `EmotionalSupportAgent` 为核心
- 检索能力：本地 KB + 飞书同步 + 飞书实时补查 + 联网搜索
- 多模型网关：DeepSeek / Doubao / OpenAI 可切换
- 记忆能力：Session Memory + User State Memory
- 前端展示：主回答 + 思考摘要 + 来源引用

当前系统更准确的产品定义是：

**受控知识增强型对话助手**

而不是：

**完全自主型多模态智能体**

---

## 2. 当前架构工作模式

### 2.1 主链路
1. 前端 `ChatPanel` 发起 `/v1/agent/chat`
2. `Agent Orchestrator` 接收请求并统一编排
3. `Safety Precheck` 执行高风险拦截
4. `Main Router Agent` 做意图识别和路由
5. 进入对应子 Agent：
   - `EmotionalSupportAgent`
   - `FeatureGuideAgent`
   - `RiskEscalationAgent`
6. 子 Agent 调用工具与 Skill：
   - 用户画像
   - 风险工具
   - 本地知识
   - 飞书同步知识
   - 飞书实时补查
   - 联网搜索
7. `Grounded Retrieval` 汇总证据包
8. `Model Gateway` 调用选定模型生成结构化回复
9. `Safety Postcheck` 做越界审查和降级
10. `Response Composer` 输出主回答、来源、思考摘要
11. 写入 `memory + audit`

### 2.2 当前知识优先级
1. 本地产品知识
2. 飞书同步知识
3. 飞书实时补查
4. 联网搜索

### 2.3 当前模型参与方式
- 模型主要负责“基于证据生成回答”
- 模型不直接决定知识来源
- 模型不拥有完全自由的工具调用权
- 风险与路由仍由规则和编排层主控

---

## 3. 当前阶段还缺的关键层

### 3.1 多模态理解层
当前以文本为主，还缺：
- 图片输入统一协议
- 语音输入统一协议
- 多模态安全审查
- 附件级结构化上下文注入

### 3.2 专业知识编排层
当前已有检索，但还缺：
- 知识分级
- 时效控制
- 冲突处理
- 医疗边界标签化
- 证据强弱分层

### 3.3 任务规划层
当前是“路由式问答编排”，还不是“可规划 Agent”。

还缺：
- 多步任务拆解
- 工具调用计划器
- 澄清与重试策略状态机
- 子任务执行状态跟踪

### 3.4 Skill 生态层
当前 skill 以检索类为主，还缺：
- 分析类 skill
- 动作类 skill
- 权限等级
- skill workflow 编排

### 3.5 个性化策略层
当前有 memory，但还缺：
- 用户偏好稳定提炼
- 阶段化状态机
- 个性化回答策略
- 个性化推荐策略
- 干预节奏控制

### 3.6 评估与反馈闭环层
当前还缺：
- 命中率评估
- 误路由分析
- 高风险漏检监控
- Prompt/Skill/知识 A/B 对比
- 用户反馈到策略回流机制

### 3.7 运营后台层
当前知识和 Prompt 主要靠文件维护，还缺：
- 知识条目管理
- Prompt 版本管理
- 模型开关与策略配置
- 风险规则管理
- 审核与发布流程

## 4. 迭代路线图

## Phase 1：知识库打底
### 目标
把“回答有据可依”做扎实，形成稳定知识底座。

### 本阶段重点
- 飞书同步链路稳定
- 飞书实时补查稳定
- 8 篇核心知识卡补齐
- 回答引用稳定可展示
- `feishu-sync-targets.local.json` 维护流程固定

### 交付物
- 首批飞书知识文档
- 同步后的本地知识快照
- 能命中、能引用、能回溯的回答链路

### 验收标准
- 核心问题能稳定命中飞书知识
- 回答带标题、链接、摘要引用
- 未命中时能平稳回退
- 高风险问题不走普通知识回答

---

## Phase 2：Prompt + Skill 强化
### 目标
把系统从“能回答”提升到“回答更像产品、风格更稳、动作更清楚”。

### 本阶段重点
- 完善剩余知识卡：
  - `APP-001`
  - `RISK-001`
- 把各场景 Prompt 做成更稳定的结构化模板
- 拓展检索类 skill 到分析类 skill
- 增强 grounded answer 约束

### 交付物
- 更完整的知识提示体系
- 结构化 Prompt Pack
- 更稳定的回答风格与降级策略

### 验收标准
- 回答风格一致性提升
- 无明显“空话”“鸡汤感”
- 知识命中后能更自然地利用证据生成回答

---

## Phase 3：多模态输入层
### 目标
让系统真正进入“多模态 Agent”入口阶段。

### 本阶段重点
- 图片输入协议
- 语音输入协议
- 多模态消息标准化
- 多模态安全过滤

### 建议优先顺序
1. 语音转文本
2. 图片识别（如食物/记录类）
3. 图片 + 文本联合上下文

### 交付物
- 统一的多模态 message schema
- 至少 1 条图片分析链路
- 至少 1 条语音输入链路

### 验收标准
- 图片/语音输入能进入现有主链路
- 不破坏现有文本回答质量
- 安全拦截规则仍然有效

---

## Phase 4：任务规划层
### 目标
从“问答路由器”升级为“会拆解任务的专业 Agent”。

### 本阶段重点
- 增加任务规划器
- 引入多步 tool plan
- 引入澄清轮次控制
- 增加 workflow 状态

### 典型场景
- 先判断是否需要更多信息
- 再决定查哪些知识
- 再决定是否调用特定 skill
- 最后再生成答案

### 交付物
- Planner / Executor 分离
- 多步任务协议
- 调试与回放能力

### 验收标准
- 复杂问题能稳定走多步编排
- 不因增加规划层而破坏确定性和可审计性

---

## Phase 5：个性化与评估闭环
### 目标
形成产品壁垒，而不是只依赖模型能力。

### 本阶段重点
- 用户偏好沉淀
- 个性化回答/推荐策略
- 误路由和低质量回答评估
- Prompt/Skill/知识命中效果对比
- 运营后台初版

### 交付物
- 个性化策略引擎
- 质量评估看板
- Prompt/Skill 配置后台

### 验收标准
- 回答更贴近用户长期偏好
- 高价值问题的命中率和满意度可被量化
- 知识更新和策略迭代不再依赖纯手工改代码

---

## 5. 优先级建议

如果只从“短期最值得做”的角度看，建议严格按下面顺序：

1. 飞书知识库补齐与稳定同步
2. `APP-001` 和 `RISK-001` 文档补齐
3. grounded answer 提示词强化
4. 语音输入接入
5. 图片理解接入
6. 任务规划层
7. 评估与后台

不要现在同时做：
- 向量库
- 全自动 workflow
- 多模态全开
- 复杂后台

先把“知识稳定命中 + 回答稳定可信”做好，会比盲目上更多模型功能更有价值。
