# 飞书知识库首批 8 篇核心包

这份文档用于帮助你在飞书里创建第一批知识库文档，并把它们登记到本地同步目标文件里。

## 1. 建议目录
- `00_产品边界`
- `01_情绪支持`
- `02_睡眠支持`
- `03_饮食建议`
- `04_运动建议`
- `05_风险升级`
- `06_App功能`
- `07_伴侣支持`

## 2. 本地目标文件
编辑这个文件，把每篇飞书文档的真实 `documentId` 和 `sourceUrl` 填进去：

- `server/data/feishu-sync-targets.local.json`

同步命令：

```bash
npm run sync:feishu-kb
```

同步后的本地知识库文件：

- `server/data/feishu-knowledge.local.json`

## 3. 每篇文档统一最小结构
每篇飞书文档都建议按下面 6 段写：

1. 这篇内容解决什么问题
2. 适用范围
3. 推荐回答要点
4. 禁止越界内容
5. 可引用的固定表述
6. 维护信息（版本 / 负责人 / 更新时间）

## 4. 8 篇文档写作重点

### PROD-001｜AI 助手简介与回答边界
- 要解决什么：说明 AI 能做什么、不能做什么、什么时候会升级到高风险处理
- 重点内容：
  - 支持的场景：情绪、睡眠、饮食、运动、功能解释、内容推荐
  - 不做的事：诊断、药物建议、疗效保证
  - 风险升级触发条件
  - 如何使用 App 内工具继续下一步

### EMO-001｜孕期情绪支持手册
- 要解决什么：用户说焦虑、压力大、烦、低落、想哭时的承接与回复策略
- 重点内容：
  - `承接 -> 建议 -> 引导` 的回复结构
  - 适合使用的共情句式
  - 可执行的小步骤
  - 不可说的话：说教、空洞鼓励、绝对承诺

### SLP-001｜孕期睡眠支持知识卡
- 要解决什么：夜醒、入睡困难、易惊醒、睡不踏实
- 重点内容：
  - 睡眠卫生建议
  - 夜醒后的微动作
  - 需要观察的异常情况
  - 不要给药物或确定性医学建议

### DIET-001｜孕期饮食建议
- 要解决什么：吃什么、能不能吃、怎么搭配、忌口和替代方案
- 重点内容：
  - 常见饮食问题与替代建议
  - 孕期饮食边界
  - 禁止处方化表达
  - 没把握时如何降级回答

### EXE-001｜孕期运动建议
- 要解决什么：能不能运动、强度如何、哪些动作更安全
- 重点内容：
  - 低风险活动建议
  - 停止条件
  - 孕周相关的保守提醒
  - 不做损伤原因判断

### RISK-001｜高风险识别与升级协议
- 要解决什么：自伤、轻生、强烈绝望等高风险输入时系统该如何处理
- 重点内容：
  - 高风险关键词与同义表达
  - 立即升级动作
  - 停止普通安抚的条件
  - 紧急联系与线下支持建议

### APP-001｜App 功能说明与入口导航
- 要解决什么：打卡在哪、CBT 怎么用、内容推荐在哪里看
- 重点内容：
  - 功能入口路径
  - 每个功能的用途
  - 常见使用问题
  - 面向新用户的最短操作路径

### PARTNER-001｜伴侣沟通与支持建议
- 要解决什么：如何和伴侣表达需求、如何减少误解和冲突
- 重点内容：
  - 可以直接引用的沟通句式
  - 表达支持需求的方法
  - 冲突升级时的降温表达
  - 不做伴侣关系诊断

## 5. 飞书文档可直接复用的统一模板

```markdown
## 这篇内容解决什么问题
一句话说明这篇内容要服务的用户问题。

## 适用范围
- 适用人群：
- 典型场景：
- 不适用场景：

## 推荐回答要点
- 要点 1
- 要点 2
- 要点 3

## 禁止越界内容
- 不做诊断
- 不给药物建议
- 不做疗效承诺
- 不给高风险场景下的普通安抚替代方案

## 可引用的固定表述
- “这更像是一个可以先观察和调整的方向。”
- “如果你愿意，我们可以先做一个低负担的小步骤。”
- “如果出现明显加重或高风险信号，建议优先联系线下专业支持。”

## 维护信息
- 版本：
- 负责人：
- 更新时间：
```

## 6. 你下一步怎么做
1. 在飞书中按 8 篇标题创建文档
2. 把每篇文档链接中的 `documentId` 填入 `server/data/feishu-sync-targets.local.json`
3. 把 `sourceUrl` 填成真实飞书链接
4. 运行 `npm run sync:feishu-kb`
5. 打开 `server/data/feishu-knowledge.local.json` 检查同步结果
